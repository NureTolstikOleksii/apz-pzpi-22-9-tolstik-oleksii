3 ОПИС ВИКОНАНОЇ РОБОТИ

3.1 Опис проєкту

Тема: Програмна система для автоматизації видачі ліків хворим людям у лікарні (HealthyHelper).
HealthyHelper – це програмна система, розроблена з метою автоматизації процесу видачі медичних препаратів пацієнтам у стаціонарних умовах. Основна ідея полягає в тому, щоб забезпечити точне та своєчасне виконання лікарських призначень за допомогою цифрових інструментів та IoT-пристроїв.
Система складається з чотирьох основних компонентів: веб-застосунку для лікарів та адміністрації, мобільного додатку для пацієнтів і медсестер, IoT-контейнера для автоматичної видачі ліків, а також серверної частини з базою даних. Лікарі можуть створювати індивідуальні плани лікування через веб-інтерфейс, після чого мед. сестри налаштовують відповідні контейнери за допомогою мобільного застосунку. Контейнер, підключений до Wi-Fi, автоматично відкриває потрібний відсік у вказаний час, що зменшує ризик помилок та покращує контроль над прийомом ліків.
Проєкт покликаний підвищити ефективність роботи медичного персоналу, зменшити вплив людського фактору та забезпечити безперервний моніторинг дотримання режиму лікування пацієнтів. HealthyHelper поєднує сучасні технології IoT, мобільної та веб-розробки, створюючи зручний і надійний інструмент для медичних закладів.

3.2 Заплановані та виконані функції

Початкова версія програмної системи HealthyHelper включає всі ключові функції для організації автоматизованого процесу видачі ліків у медичних установах. Основна увага приділялася створенню ефективного та злагодженого взаємозв’язку між веб-застосунком, мобільним клієнтом, серверною частиною та IoT-пристроєм. Нижче наведено перелік запланованих функцій, більшість із яких були успішно реалізовані в межах першого релізу.
1) Серверна частина
Реалізовано:
- REST API для взаємодії мобільного та веб-застосунків;
- PostgreSQL-база даних для зберігання користувачів, пацієнтів, призначень, графіків прийому;
- система авторизації та аутентифікації з розмежуванням ролей: адміністратор, лікар, медсестра, пацієнт;
- функціонал для адмін-панелі: керування користувачами, пацієнтами, контейнерами, генерація звітів та резервне копіювання.
2) Мобільний застосунок (Android, Kotlin)
Реалізовано для мед. сестри:
- перегляд списку пацієнтів із персональними даними;
- робота з контейнерами: перегляд, підключення, заповнення відсіків;
- сповіщення про пропущені прийоми;
- перегляд призначень та можливість закріпити контейнер;
- перегляд статистики прийому пацієнтів.
Реалізовано для пацієнта:
- перегляд поточного плану лікування;
- історія лікування з інформацією про лікаря та дати призначень;
- отримання сповіщень у разі пропуску прийому;
- перегляд та редагування власного профілю.
3) Веб-застосунок (React)
Функціонал для лікарів:
- пошук і сортування пацієнтів;
- додавання, редагування та видалення пацієнтів;
- створення призначень із зазначенням препарату, дози, частоти, тривалості;
- формування звіту про лікування.
Функціонал для адміністратора:
- управління медичним персоналом;
- перегляд усіх пацієнтів у системі;
- перегляд статистики по IoT-пристроях;
- перегляд загальної інформації по медичному закладу;
- створення резервних копій бази.
4) IoT-пристрій (контейнер)
Реалізовано:
- автоматичне відкриття відсіку у зазначений час;
- підключення до серверу через Wi-Fi;
- взаємодія з мобільним застосунком для налаштування та заповнення;
- відправка сигналів про відкриття, пропуски або збої;
- OLED-дисплей, серводвигуни, кнопка та базер для взаємодії з пацієнтом.
Контейнер підтримує шість відсіків, кожен з яких відповідає окремому прийому. Заповнення виконується щодня мед. сестрою відповідно до призначень, зазначених у мобільному застосунку.
Таким чином, реалізовано повноцінну програмну систему HealthyHelper, яка об'єднує веб-застосунок, мобільний додаток, серверну частину та IoT-пристрій для автоматизації процесу видачі ліків у лікарнях. Система забезпечує взаємодію між усіма ролями користувачів, дозволяє ефективно управляти призначеннями та контролювати прийом препаратів, що підвищує точність лікування та зменшує навантаження на медичний персонал.
